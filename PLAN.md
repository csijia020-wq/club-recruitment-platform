# 社团招新智能匹配平台 PLAN

## 1. 产品目标
聚焦高校新生社团招新场景，直击传统模式下 **“信息不对称、报名流程繁琐、匹配效率低下、进度不透明”四大痛点** 。

致力于构建一个一站式招新管理平台，无缝串联“学生探索发现”与“社团筛选录取”的双向互选流程。引入轻量级 AI 赋能推荐机制，确保 V1 版本在具备智能化的同时，保持高度的透明性与可解释性。

## 2. 核心判断
本产品首先是一款聚焦于“提升匹配效率与报名转化率”的效能工具，而非泛泛的校园社交社区。

- 对学生（C端）：追求极致的体验闭环，“看得全、看得懂、投得快、能跟踪”。
- 对社团（B端）：追求极致的管理效率，“配置标准化、筛选高效化、通知自动化、数据可视化”。
- 对学校（G端）：追求极致的治理规范，“审核统一、节奏统一、数据统一”。

## 3. 用户角色与功能模块
### 3.1 学生端（侧重体验与转化）
- 智能迎新：通过引导式交互，采集 MBTI、兴趣标签、专业技能、时间偏好及自我介绍，构建初始用户画像，破除冷启动困境。
- 发现与推荐：以卡片流或列表流形式呈现社团，并高亮展示“推荐理由”，降低信息噪音与选择焦虑。
- 社团主页：结构化展示社团定位、活动风采、招新要求、时间投入预期及往届评价，升级为多维度的可决策信息。
- 极简报名：自动复用基础画像资料，学生仅需填写该社团的专属自定义问题，大幅降低重复填表成本。
- 进度追踪：统一看板展示所有投递状态（已提交 / 审核中 / 一面 / 二面 / 已录取 / 婉拒），消除等待期的信息黑洞。
- 通知中心：聚合站内信与邮件及短信提醒，确保关键流程节点触达率 100%。

### 3.2 社团管理端（侧重提效与协同）
- 招新配置：自定义招新周期、名额限制、人群标签偏好及专属报名表单，将规则收敛至标准化后台。
- 申请管理看板：采用看板模式，按流转状态可视化管理候选人，提升简历筛选与流转效率。
- 智能触达：支持按状态批量发送面试邀约、材料补充及录取或婉拒通知，解放繁琐的机械性沟通。
- 数据洞察：实时展示报名趋势、专业分布、状态流转及转化漏斗，沉淀招新数据资产。

### 3.3 系统管理端（侧重合规与统筹）
- 社团准入：审核新社团入驻资质及存量社团年度注册，把控平台内容质量。
- 全局看板：鸟瞰全校社团生态，包括类型分布、整体热度及招新转化大盘，提供宏观决策支持。
- 活动治理：统一发布招新日历及系统级治理提醒，确保全校招新节奏步调一致。

## 4. 核心数据对象
- 学生画像：姓名、专业、年级、MBTI、兴趣标签、技能标签、可用时长、自我介绍、核心诉求。
- 社团画像：机构介绍、分类标签、准入要求、品牌活动、往届评价、时间投入预期、富媒体物料。
- 招新表单：系统通用字段、社团自定义字段、附件模块、提交校验规则。
- 申请流转：投递记录、当前节点状态、状态更新时间戳、社团内部备注、触达通知日志。
- 匹配度引擎：综合匹配得分、得分权重构成、白盒化推荐理由、解释性文本。

## 5. AI 能力设计
- 多维标签匹配：基于兴趣、技能、时间偏好及社团属性标签，构建基础匹配度算法模型。
- 可解释性推荐：前端明确展示“为什么推荐你”，建立用户信任。
- 轻量级语义增强：提取学生自我介绍与社团描述的关键词，通过共现分析进行匹配度加权。
- 后续扩展：深度语义相似度匹配、招新智能问答、报名自荐信辅助写作。

## 6. MVP 版本边界
- 核心聚焦：
  - 学生端：画像采集、智能推荐、社团主页、一键报名、状态追踪看板。
  - 社团端：招新规则配置、看板式候选人管理、批量通知触达、基础数据统计。
  - 学校端：社团资质审核、全局数据概览、治理日历发布。
  - 底层能力：标签匹配引擎与白盒化推荐解释。
- 明确不碰：
  - 大而全的校园社交社区。
  - 即时通讯聊天功能、直播宣讲功能。
  - 多学校、多租户的复杂底层架构。
  - 高风险的黑盒 AI 自动录取或淘汰决策。

## 7. 实施路线图
- Phase 0 架构基建：明确信息架构与页面地图，固化核心数据对象与底层状态机流转逻辑。
- Phase 1 学生端闭环：优先跑通 C 端主流程（画像采集 → 智能推荐 → 浏览主页 → 极简报名 → 进度追踪）。
- Phase 2 社团端赋能：上线社团管理后台，打通规则配置、简历筛选、状态推进与批量通知链路。
- Phase 3 校方统筹与全局数据：交付学校管理端，实现统一的社团审核与招新大盘数据可视化。
- Phase 4 AI 深度演进：引入真实业务数据进行模型调优，迭代 AI 语义增强与推荐策略。

## 8. 验收标准
1. 极速启动：新生能在 3 分钟内完成画像采集，并顺滑获取首批个性化推荐列表。
2. 推荐透明：所有推荐结果均附带清晰的推荐解释文案，而非单一的分数。
3. 无感填表：学生在报名多个社团时，基础资料实现 100% 自动带入，无重复填写动作。
4. 状态透明：投递完成后，学生能实时查看进度变更，并在关键节点收到准确的系统提醒。
5. 高效流转：社团负责人能够完全脱离线下表格，仅通过系统看板即可完成全流程的管理与通知。
6. 全局掌控：学校管理员能顺畅完成社团年审，并实时调取全校招新进度与转化数据。
